# KnowledgeExporter 使用指南

将 Cursor 开发过程中积累的知识资产导出为独立文档，确保团队经验不因工具切换而丢失。

---

## 工具包内容

```
KnowledgeExporter/
├── 使用指南.md                       ← 本文件
├── cursor-knowledge-export-checklist.md  ← 人工参考版清单（可打印/离线用）
├── skill/
│   ├── SKILL.md                      ← Skill：主动扫描+导出
│   └── references/
│       └── asset-checklist.md        ← 知识资产清单（Skill/Rule 共享数据源）
└── rule/
    └── knowledge-recording.mdc       ← Rule：日常自动提醒记录
```

---

## 快速开始（3 步）

### 第 1 步 · 安装到 Cursor

**安装 Skill**（全局生效，所有项目可用）：

```
将 skill/ 整个文件夹复制到：
  Windows: %USERPROFILE%\.cursor\skills\knowledge-export\
  macOS:   ~/.cursor/skills/knowledge-export/

最终目录结构：
  .cursor/skills/knowledge-export/
  ├── SKILL.md
  └── references/
      └── asset-checklist.md
```

**安装 Rule**（需要在每个项目中安装）：

```
将 rule/knowledge-recording.mdc 复制到目标项目的：
  <项目根目录>/.cursor/rules/knowledge-recording.mdc
```

### 第 2 步 · 主动导出（Skill）

在 Cursor 对话中说以下任意一句：

| 你说 | AI 做什么 |
|------|----------|
| "帮我导出知识资产" | 扫描项目 → 识别缺失文档 → 逐项引导生成 |
| "检查文档完整性" | 扫描已有文档 → 对照清单 → 列出缺失项 |
| "检查编码阶段的文档" | 仅扫描编码阶段（③）的相关文档 |
| "帮我补充编译指导文档" | 直接生成指定文档 |

AI 会自动执行：
1. 扫描项目 `doc/` 目录，识别已有文档
2. 对照 6 个研发阶段的清单，找出缺失项
3. 让你选择优先导出哪些
4. 逐项生成文档，保存到 `doc/` 目录

### 第 3 步 · 日常自动提醒（Rule）

安装 Rule 后无需任何操作。当你在日常开发中：

- 修完一个 bug → AI 提醒你记录分析过程
- 完成一个需求 → AI 提醒你记录设计决策
- 改了编译配置 → AI 提醒你更新编译指导
- 做了性能优化 → AI 提醒你记录基线数据

提醒格式示例：
```
💡 知识资产提醒：刚才修复的 [信号槽连接类型bug] 建议记录到 doc/历史bug分析记录.md
   需要我现在帮你整理吗？（回复"记录"即开始，忽略即跳过）
```

回复"记录"，AI 会用内置模板帮你快速生成文档。忽略则跳过，不影响工作。

## 使用场景

### 场景 A · 紧急导出（截止日期前批量导出）

适合：临近 Cursor 停用截止日期，需要集中导出。

1. 打开目标项目的 Cursor 对话
2. 说："帮我导出知识资产"
3. AI 扫描后展示缺失清单，按优先级选择导出
4. 建议按 P0→P1→P2 顺序逐阶段完成

> 小技巧：如果一个项目有多个 Cursor 对话（不同时期），建议在每个对话中都执行一次扫描，因为不同对话有不同的上下文记忆。

### 场景 B · 日常积累（长期使用）

适合：希望在日常工作中持续积累文档。

1. 安装 Rule 到项目
2. 正常工作，AI 会在合适时机提醒
3. 觉得有价值就回复"记录"，没价值就忽略
4. 定期（如每周五）说"检查文档完整性"做一次全面自检

### 场景 C · 新项目初始化

适合：新项目开始时就建立文档习惯。

1. 安装 Skill + Rule
2. 说："帮我初始化项目文档框架"
3. AI 会根据项目类型创建 `doc/` 目录结构
4. 后续通过 Rule 的日常提醒持续补充

## 如何扩展清单

当你发现有知识资产不在现有清单中时，有两种方式补充：

### 方式一 · 让 AI 自动补充

在 Cursor 中说：
```
我发现还需要记录"XX内容"，请补充到知识资产清单中
```
AI 会自动将新项追加到 `references/asset-checklist.md` 的对应阶段。

### 方式二 · 手动编辑

打开 `skill/references/asset-checklist.md`，在对应阶段的表格末尾追加一行：

```markdown
| 3.8 | 新的资产名称 | 说明文字 | 参考提示词文本 |
```

追加后，Skill 下次扫描时会自动包含新项。

## 团队推广建议

### 分发方式

1. 将整个 `KnowledgeExporter/` 文件夹放到团队共享目录（如 Git 仓库、共享网盘）
2. 每个成员按「第 1 步」安装到自己的 Cursor
3. 团队成员在自己的每个项目中执行导出

### 推广顺序

| 阶段 | 时间 | 动作 |
|------|------|------|
| 第 1 周 | 即刻 | 分发工具包，每人安装 Skill + Rule |
| 第 1-2 周 | 本周 | 每人在自己的项目中执行"导出知识资产"，优先导出 P0 项 |
| 第 2-3 周 | 下周 | 补充 P1、P2 项，交叉检查文档质量 |
| 长期 | 持续 | Rule 自动提醒，保持日常记录习惯 |

### 质量检查

导出完成后，用以下标准验证每份文档：

- [ ] 新人仅凭此文档能独立完成对应工作
- [ ] 操作步骤可直接执行，不含模糊描述
- [ ] Bug 记录有完整链路：现象 → 分析 → 根因 → 修复 → 验证
- [ ] 文档内容不依赖 Cursor 对话（这些将不可访问）

## 常见问题

**Q: Skill 和 Rule 有什么区别？**
- Skill 是**主动工具**：你说"导出知识资产"时触发，做系统性扫描和文档生成
- Rule 是**被动提醒**：日常开发中自动检测，在合适时机温和提醒你记录

**Q: 一个项目有多个 Cursor 对话，怎么办？**
- 在每个有价值的对话中分别执行"导出知识资产"，不同对话有不同的上下文

**Q: AI 生成的文档不准确怎么办？**
- 导出后务必人工审查。AI 生成的是初稿框架，细节需要你确认和修正

**Q: 如果项目从未用 Cursor 做过某个阶段的工作？**
- 扫描时该阶段的项会显示"➖ 不涉及"，直接跳过即可

**Q: 安装 Rule 后提醒太频繁怎么办？**
- Rule 设计为不打断工作流程（任务完成后才提醒），且可以忽略
- 如果确实不需要，可以删除项目中的 `.cursor/rules/knowledge-recording.mdc`
